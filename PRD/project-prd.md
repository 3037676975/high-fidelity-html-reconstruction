# 高保真截图转 HTML 页面 PRD

## 1. 项目概述

本项目目标是根据用户提供的 CodePath / GitHub 基础操作指南截图，使用 `high-fidelity-html-reconstruction` 工作流，将视觉稿高保真还原为可访问、可部署的 HTML 静态页面。

项目最终页面需要在桌面端尽可能接近参考截图，并支持基础移动端适配。页面已部署至 Vercel，便于用户直接访问与分享。

## 2. 项目目标

1. 将上传截图还原为静态 HTML 页面。
2. 保持页面主要视觉结构、色彩氛围、卡片层次、图标风格和内容布局接近参考图。
3. 使用绿色背景源图流程提取资产，避免直接裁切原始截图作为页面主体。
4. 修复页面在下滑、低宽度、局部模块中的重叠和错位问题。
5. 完成线上部署并提供可访问域名。

## 3. 目标用户

- 需要将 UI 截图快速转成 HTML 原型的用户。
- 需要展示「GitHub 基础操作指南」页面视觉效果的用户。
- 后续希望复用高保真还原流程进行更多页面重建的维护者。

## 5. 视觉与交互要求

### 5.1 视觉风格

- 深色科技感背景。
- 蓝紫渐变高光。
- 半透明玻璃卡片。
- 细描边、弱阴影、多层背景光效。
- 图标与按钮保持圆角、发光和渐变质感。

### 5.2 布局要求

- 桌面端参考宽度约为 941px。
- 左侧导航固定在页面左栏视觉区域，但不应在滚动时覆盖下方模块。
- 主内容与参考图保持接近的上下节奏。
- `常用命令速查`、`实战流程`、`FAQ` 等下方模块需要向左展开，贴近参考截图中的完整宽度视觉。

### 5.3 细节要求

- 表单组件中的文字不能溢出或重叠。
- 命令卡片中的代码不能撑破卡片。
- FAQ 问题、说明文字和箭头不能相互覆盖。
- 页面下滑时左侧导航不能压住下方内容。
- 移动端需要避免横向溢出和布局错乱。

## 6. 技术方案

### 6.1 技术栈

- HTML
- CSS
- JavaScript / Node.js
- `pngjs` 用于图片处理
- Vercel 静态部署

### 6.2 核心流程

1. 用户上传参考截图。
2. 使用 GPT img2img 生成完整绿色背景源图 `green_source.png`。
3. 使用脚本提取绿色背景源图中的独立素材。
4. 生成 `assets/` 和 `asset_manifest.json`。
5. 使用 HTML/CSS 重建页面结构和视觉样式。
6. 使用浏览器截图进行视觉验证。
7. 迭代修复布局、重叠、错位和移动端适配问题。
8. 部署到 Vercel。

## 7. 当前交付物

## 10. 验收标准

1. 页面能在本地直接打开并正常显示。
2. 页面能在线上 Vercel 地址访问。
3. 页面主要视觉结构与参考截图一致。
4. 主要素材来自绿色背景源图提取结果。
5. 桌面端无明显错位、遮挡和文字重叠。
6. 移动端无严重横向溢出。
7. 下滑页面时左侧导航不会覆盖下方模块。
8. 表单、FAQ、命令卡片等细节模块无文字重叠。
